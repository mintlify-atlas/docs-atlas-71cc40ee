## Overview

Deserializes a JSON-formatted profile string into a `Profile` object. This function is designed for transaction-based profiles and supports platform-specific deserialization for improved performance.

## Signature

```python
def profile_from_json_str(
    profile: str,
    platform: Optional[str] = None
) -> Profile
```

## Parameters

<ParamField path="profile" type="str" required>
  A profile serialized as a JSON string.
</ParamField>

<ParamField path="platform" type="Optional[str]">
  An optional string representing the profile platform (e.g., "python", "node", "android"). If provided, the function can directly deserialize to the right profile more efficiently. If the platform is known at the time this function is invoked, it's recommended to always pass it.
</ParamField>

## Returns

<ResponseField name="return" type="Profile">
  A `Profile` instance containing the deserialized profile data.
</ResponseField>

## Raises

<ResponseField name="ValueError" type="Exception">
  Raised if an error occurs during the deserialization process, such as invalid JSON format or corrupted profile data.
</ResponseField>

## Example

```python
import vroomrs
import json

# Load profile JSON from file
with open("transaction_profile.json", "r") as f:
    profile_json = f.read()

# Deserialize without platform hint
profile = vroomrs.profile_from_json_str(profile_json)

# Deserialize with platform hint for better performance
profile = vroomrs.profile_from_json_str(profile_json, platform="python")

# Access profile data
print(f"Profile ID: {profile.get_profile_id()}")
print(f"Platform: {profile.get_platform()}")
print(f"Duration: {profile.duration_ns()}ns")

# Extract function metrics
metrics = profile.extract_functions_metrics(
    min_depth=1,
    filter_system_frames=True,
    max_unique_functions=10
)

for metric in metrics:
    print(f"Function: {metric.get_function()}, Self Time: {metric.get_sum_self_time_ns()}ns")
```

## Notes

- For optimal performance, always provide the `platform` parameter when the platform is known
- This function is specifically designed for transaction-based profiles
- For continuous profiling chunks, use [`profile_chunk_from_json_str`](/api/profile-chunk-from-json-str) instead
- The function validates the JSON structure and will raise an error if the format is invalid
- After deserialization, you may want to call `profile.normalize()` to apply platform-specific normalization
